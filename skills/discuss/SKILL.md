# 方案讨论

轻量级讨论流程，比 brainstorming 少 70% 的步骤。目标：**想清楚再动手**。

## 流程

### 第一步：理解问题

通过 2-3 轮追问理清需求和约束。每次只问一个问题，优先用选择题。

需要搞清楚的事情：
- 要解决什么问题？背景是什么？
- 有什么约束？（技术栈、时间、兼容性等）
- 成功的标准是什么？

### 第二步：方案对比

给出 2-3 个可行方案，每个方案说明：

| 维度 | 方案A | 方案B | 方案C |
|------|-------|-------|-------|
| 思路 | ... | ... | ... |
| 优点 | ... | ... | ... |
| 缺点 | ... | ... | ... |
| 复杂度 | 低/中/高 | 低/中/高 | 低/中/高 |

最后给出推荐方案和理由，然后主动询问：

> "以上方案你倾向哪个？确认后我生成讨论文档。也可以：继续讨论 / 换方向 / 直接退出不保存"

- 用户选择某个方案 → 进入第三步生成文档
- 用户要继续讨论 → 回到方案对比继续
- 用户要退出 → 不落盘，结束讨论

### 第三步：沉淀结论

用户确认方向后，生成讨论文档到 `docs/discussions/`，包含：

1. **问题背景** — 要解决什么
2. **方案对比** — trade-off 表格
3. **最终决定** — 选的哪个方案、为什么
4. **后续步骤** — 接下来做什么

## 文档约定

- 文件命名: `docs/discussions/YYYY-MM-DD-<topic>.md`
- 每次讨论独立一个文件
- 生成文件后，更新 `docs/.vitepress/config.mts` 侧边栏配置，将新文档注册到 discussions 分组

## 原则

- **一次只问一个问题** — 不连珠炮
- **选择题优先** — 降低用户输入成本
- **YAGNI** — 只讨论当前问题，不扩展
- **就事论事** — 不确定不急着写文档，确认后才落盘
- **纯 Markdown** — 文档不用任何 VitePress 特有语法
