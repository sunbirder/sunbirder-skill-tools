# 项目文档生成

深入扫描项目代码，自动生成一套丰富完整的开发文档。

## 流程

### 第一步：项目分析

全面扫描项目，理解其全貌：

- `package.json` — 依赖、脚本、项目元信息 → 推断项目类型（CLI/库/Web 应用/工具集）
- 目录树 — 各目录职责
- 入口文件 — 主流程
- 源码 — 核心模块、导出 API、关键函数
- 配置文件 — 构建、环境、部署相关
- CLAUDE.md — 已有项目说明
- README.md — 已有文档
- `git log` — 近期改动趋势

### 第二步：确定文档结构

根据项目类型，提出文档清单让用户确认。不同项目类型的文档侧重：

**CLI 工具型：**
- 项目概述、安装指南、命令参考、配置说明、开发指南、架构说明

**库/SDK 型：**
- 项目概述、快速开始、API 参考、使用示例、配置说明、开发指南

**Web 应用型：**
- 项目概述、架构说明、页面路由说明、组件文档、部署指南、开发指南

### 第三步：逐份生成

按优先级逐份撰写，每份文档要求：

- 有实际代码引用（文件路径 + 行号）
- 有可运行的代码示例
- 有清晰的目录结构
- 有架构图或流程图（Mermaid，如适用）

每份生成后向用户确认，再继续下一份。

### 第四步：注册到文档站

所有文档生成完成后，更新 `docs/.vitepress/config.mts` 侧边栏配置，将新文档注册到对应的导航分组。

## 原则

- **以代码为准** — 文档内容基于实际代码，不凭空编造
- **优先检测已有文档** — 如果文件已存在，询问是否覆盖还是补充
- **丰富但不冗余** — 每个文档有价值，不只堆砌
- **引用代码** — 关键描述带上文件路径和行号
