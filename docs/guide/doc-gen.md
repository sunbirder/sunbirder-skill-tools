# doc-gen

根据项目代码自动生成完整丰富的开发文档。

## 触发条件

- 输入 `/skill:doc-gen`
- 说 "生成项目文档" 或 "帮我给这个项目写文档"

## 流程

1. **项目分析** — 扫描 package.json、目录树、源码、配置，推断项目类型
2. **确定文档结构** — 根据项目类型提出文档清单，用户确认
3. **逐份生成** — 每份文档包含代码引用、可运行示例、架构说明
4. **注册到文档站** — 更新 VitePress 侧边栏

## 项目类型与文档侧重

| 项目类型 | 生成的文档 |
|----------|-----------|
| CLI 工具 | 概述、安装、命令参考、配置、开发指南、架构说明 |
| 库/SDK | 概述、快速开始、API 参考、示例、配置、开发指南 |
| Web 应用 | 概述、架构、路由说明、组件文档、部署、开发指南 |

## 文档质量标准

- 有实际代码引用（文件路径 + 行号）
- 有可运行的代码示例
- 有清晰的目录结构
- 有架构图或流程图（Mermaid）

## 与 docs-sync 的区别

| 维度 | doc-gen | docs-sync |
|------|---------|-----------|
| 场景 | 新建文档、从头生成 | 更新已有文档 |
| 范围 | 全项目 | 按需/增量 |
| 产出 | 完整文档集 | 修正后文档 |
