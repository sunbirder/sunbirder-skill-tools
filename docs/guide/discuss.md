# discuss

轻量方案讨论 — 对比选项、确认方向、沉淀结论。

## 触发条件

- 输入 `/skill:discuss`
- 说 "讨论一下 xxx 方案" 或 "帮我分析一下这几个方案"

## 流程

1. **理解问题** — 2-3 轮追问理清需求和约束
2. **方案对比** — 给出 2-3 个可行方案，表格对比 trade-off
3. **沉淀结论** — 用户确认后生成 `docs/discussions/YYYY-MM-DD-<topic>.md`

## 设计原则

- 一次只问一个问题，优先选择题
- 只讨论当前问题，不扩展（YAGNI）
- 确认后才落盘，不确定不写文档

## 产出

```
docs/discussions/
├── index.md                       # 讨论记录首页
└── YYYY-MM-DD-<topic>.md          # 讨论文档
```

每份讨论文档包含：
- 问题背景
- 方案对比表格
- 最终决定和理由
- 后续步骤

## 与 brainstorming 的区别

| 维度 | brainstorming | discuss |
|------|---------------|---------|
| 适用场景 | 复杂功能、多子系统 | 小修改、方案讨论 |
| 流程步骤 | 9 步 | 3 步 |
| 产出 | 设计文档 + spec | 讨论结论 |
| 追问轮次 | 多轮 | 2-3 轮 |
| 文档格式 | spec 规范 | 自由 Markdown |
