# docs-sync

项目持续开发过程中，文档与代码对齐。

## 触发条件

- 输入 `/skill:docs-sync`
- 说 "更新一下文档" 或 "文档和代码对齐一下"

## 流程

1. **确定范围** — 全部文档还是指定部分，以 `git log` 为线索
2. **扫描代码** — 读取实际代码，记录当前状态
3. **对比文档** — 逐项对照，列出过时/遗漏/已删除的问题
4. **确认后修复** — 用户 review 问题清单，逐条确认后修改

## 原则

- 以代码为准，文档服务于代码
- 最小改动，只修问题不重写
- 每条修改标注代码出处
- 先确认再改
